# Projects

This page collects the projects that best represent what I am building and learning now.

I am not trying to list every repository I have ever created. The goal is to show a clearer line of work: AI workflows, agentic product engineering, creator tools, personal publishing, and selected frontend engineering experiments.

## Current focus

- Turning AI workflows into reusable tools
- Building agent workstations for writing, research, and creator operations
- Exploring AI-assisted content operation systems for creators and small teams
- Keeping a personal publishing pipeline for long-form writing and ebooks
- Using frontend engineering experience to ship practical AI products

## Featured projects

### Agent Workstation

A reusable multi-agent orchestration framework for building task-specific AI workstations.

It is designed around agents, YAML pipelines, schema validation, retries, parallel execution, and run artifacts. The idea is simple: instead of relying on one long prompt, turn a complex workflow into explicit, reusable steps.

- GitHub: <https://github.com/XingMXTeam/agent-workstation>
- Direction: AI workflow infrastructure, agentic engineering, task-specific workstations

### Xiaohongshu AI Operator

An AI content operation system for Xiaohongshu creators, personal brands, local businesses, and small teams.

The goal is not just AI writing. The goal is a repeatable content operating system: account profile, competitor research, topic pool, note brief, draft generation, human review, publishing plan, and feedback loop.

- GitHub: <https://github.com/XingMXTeam/xiaohongshushipfast>
- Direction: creator tools, AI content operations, SEO/GEO, small-team automation

### MCP Screenshot Inverter

A small MCP + Puppeteer experiment that lets an LLM call browser tooling to capture and transform webpage screenshots.

It is intentionally small, but the pattern matters: AI should be able to call real tools, inspect real pages, and create useful artifacts instead of only producing text.

- GitHub: <https://github.com/XingMXTeam/a-simple-mcp>
- Direction: MCP, browser automation, AI tool use

### RAG Search API

A compact FastAPI service that combines web search, vector reranking, detail fetching, and content filtering into one RAG-oriented search endpoint.

This is a backend experiment for search-enhanced AI workflows.

- GitHub: <https://github.com/XingMXTeam/rag-search>
- Direction: RAG backend, search, reranking, AI information retrieval

### Ebook Factory

A Markdown-to-ebook production pipeline based on Pandoc, Make, Docker, and a TypeScript CLI.

This project supports a repeatable personal publishing workflow from draft to PDF / EPUB / MOBI. The current private repository may contain real book material, so a clean public template should be extracted later.

- GitHub: <https://github.com/XingMXTeam/ebook-maker>
- Direction: personal publishing, content products, ebook automation

### Frontend Map

A historical frontend knowledge archive covering JavaScript, TypeScript, Vue, and broader frontend engineering topics.

This is not my current flagship direction, but it shows the frontend engineering background behind my AI product work.

- GitHub: <https://github.com/XingMXTeam/frontend-map>
- Direction: frontend engineering, knowledge archive, JavaScript / TypeScript
