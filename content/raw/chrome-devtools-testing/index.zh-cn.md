本文基于项目中 Web 自动化与 Chrome 扩展自动化的实践，总结一套「Chrome DevTools + MCP」的自动化测试思路与流程。

---

## 一、环境与连接

### 1.1 前置条件

| 依赖项 | 版本/要求 | 用途 |
|--------|-----------|------|
| Chrome 浏览器 | 最新稳定版 | 被测环境 |
| Node.js | > 22 | 运行 MCP 与脚本 |
| Chrome DevTools MCP | 最新版 | 提供浏览器自动化 API |

### 1.2 连接 Chrome DevTools MCP

```json
"chrome-devtools": {
    "command": "npx",
    "args": [
        "-y",
        "chrome-devtools-mcp@latest",
        "--autoConnect"
    ],
    "env": {}
}
```

---

## 二、两类自动化场景

1. **Web 页面自动化**：普通网站的登录、菜单导航、接口/数据校验。
2. **Chrome 扩展自动化**：扩展的 popup/onboarding 页面、扩展页检查、扩展内业务流程。

---

## 三、Web 页面自动化流程要点

1. **浏览器环境初始化**
2. **访问目标网站**
3. **用户认证**（滑块验证需曲线轨迹、800-1500ms 时长模拟人类滑动）
4. **业务操作**
5. **接口校验**（如 JSON-RPC 断言）

**行为模拟**：输入每字间隔 50-150ms，操作间隔 300-800ms，页面加载后再等 1-2 秒。

---

## 四、Chrome 扩展自动化流程要点

1. **浏览器与扩展准备**
2. **插件状态检查**（`chrome://extensions/`）
3. **登录态与解锁**（popup -> onboarding -> popup 流程）
4. **业务操作**
5. **结果校验**

---

## 五、通用实践建议

1. **先连接再操作**：通过 MCP 列出标签页确认状态
2. **分阶段、分分支**：初始化 -> 认证 -> 业务 -> 校验
3. **人类化操作**：固定延迟、曲线滑动、逐字输入
4. **配置与输出结构化**：URL、账号、扩展 ID 放入配置
5. **异常表格化**：为每类场景列异常处理表
