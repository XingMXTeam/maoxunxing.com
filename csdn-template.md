# AI 你需要知道的

## 一、AI 认知升级

### 1.1 概念层级关系

```
机器学习 ⊃ 人工神经网络 ⊃ 深度学习 ⊃ 大模型
```

这是一个包含关系，每一层都是前一层的子集和进化。

### 1.2 机器学习基础

#### 核心公式
```
f(x) = wx
```
- **w**: 参数（权重），训练模型的目的就是得到合理的参数分配
- **x**: 特征，命中为 1，未命中为 0
- **特征工程**: 将各种特征转换为数字表示

#### 关键概念

| 概念 | 说明 |
|------|------|
| **样本** | 范例数据，用于训练模型 |
| **Label** | 样本的结果标签 |
| **监督学习** | 有 label 的训练方式 |
| **无监督学习** | 无 label 的训练方式 |

#### 模型预训练流程

1. **准备样本** → 收集资料
2. **特征工程** → 得到 x
3. **设计模型** → f(x) = wx（学习计划）
4. **训练模型** → 得到 y（学习过程）
5. **测评验证** → 评估效果

### 1.3 文本特征提取

#### One-Hot 向量
```
词汇表 5 万词，"苹果"是第 3 个词
→ 向量为 [0, 0, 1, 0, ..., 0]（长度 5 万，仅第 3 位为 1）
```

#### Embedding（嵌入向量）
- 本质也是向量，是对 One-Hot 的优化
- 不再只用 0 和 1，具有**语义表示**能力
- **压缩了向量维度**，更高效

### 1.4 模型任务类型

| 任务类型 | 输出特点 | 示例 |
|---------|---------|------|
| **回归** | 连续数值（不可枚举） | 房价预测、温度预测 |
| **分类** | 离散标签（可枚举） | 垃圾邮件识别、图像分类 |

### 1.5 模型训练核心概念

- **拟合**: 计算结果与标注结果一致
- **泛化能力**: 好的模型应该具备的能力
- **过拟合**: 模型记住了训练数据，但无法处理新数据
- **欠拟合**: 模型太简单，连训练数据都学不好
- **超参数**: 人工干预的参数（学习轮次、批次大小、学习率等）

#### 典型学习过程
```
数据分批(batch) → 学习 → 测试验证(loss) → 优化算法调整参数 → 继续学习
→ 完成所有数据(1 epoch) → 重复多轮直到收敛
```

#### 数据集划分
- **训练集**: 用于模型学习
- **验证集**: 用于调整超参数
- **测试集**: 用于最终评估

#### 验证方法
- **K-Fold 验证**: 交叉验证技术
- **混淆矩阵**: 判定模型是否通过测试
- **核心指标**: Accuracy, F1-Score, AUC

## 二、深度学习深入

### 2.1 激活函数（Activation Function）

> **激活函数就像"智能开关"**  
> 决定神经元是否要把接收到的信号传递出去

#### 为什么需要激活函数？

1. **加入非线性**: 没有激活函数，神经网络只能处理线性问题（如直线分割）
2. **控制输出范围**: 限制输出在特定区间

#### 常见激活函数对比

| 激活函数 | 输出范围 | 特点 | 使用场景 |
|---------|---------|------|---------|
| **Sigmoid** | 0~1 | 类似概率，易梯度消失 | 二分类输出层 |
| **ReLU** | [0, +∞) | 简单高效，最常用 | 隐藏层首选 |
| **Tanh** | -1~1 | 零中心化 | 需要负值的场景 |

**ReLU 示例**:
```python
输入 10 → 输出 10
输入 -5 → 输出 0
```

**形象类比**:  
神经网络是水管系统，激活函数就是阀门——有的只允许单向水流（ReLU），有的可调节大小（Sigmoid），使系统灵活适应复杂数据。

### 2.2 MLP（多层感知机）

```
输入层 → 隐藏层 → 输出层
```

**示例**: 输入猫的图片 → 隐藏层分析耳朵、胡须等特征 → 输出层判断"是猫"

- 加入**隐藏层和激活函数**
- 突破线性限制，能拟合更复杂函数

### 2.3 表征学习

| 类型 | 方法 | 数据特点 |
|------|------|---------|
| **图片表征** | CNN | N×M 数据 |
| **文本表征** | Word2Vec | N×1 数据，滑动窗口提取 token |

#### Word2Vec 窗口机制

- **窗口大小**: 2N+1（如 N=2，窗口大小为 5 个词）

**CBOW 模式（连续词袋）**:
- **输入**: 上下文词（2N 个）
- **输出**: 预测中心词
- **处理**: 上下文词向量取平均

**示例**:
```
句子："今天/天气/晴朗/适合/散步"
窗口大小 5，中心词"晴朗"
输入: ["今天", "天气", "适合", "散步"]
目标: 预测"晴朗"
```

## 三、如何写好提示词（Prompt Engineering）

### 3.1 核心原则

1. **明确具体**: 清晰描述任务和目标
2. **提供上下文**: 给出足够的背景信息
3. **结构化表达**: 使用分段、列表等格式
4. **示例引导**: 提供输入输出示例
5. **迭代优化**: 根据结果不断调整

### 3.2 Prompt 模板

```markdown
## 角色设定
你是一个[专业领域]专家

## 任务描述
请完成以下任务：[具体任务]

## 上下文信息
- 背景：[相关信息]
- 约束：[限制条件]

## 输出要求
- 格式：[期望格式]
- 长度：[字数限制]
- 风格：[语言风格]

## 示例
输入：[示例输入]
输出：[示例输出]
```

### 3.3 高级技巧

- **思维链（Chain of Thought）**: "请逐步思考..."
- **少样本学习（Few-Shot）**: 提供 3-5 个示例
- **角色扮演**: "你是一位资深工程师..."
- **分步执行**: "第一步...第二步..."

## 四、如何高效使用 AI

### 4.1 工作流设计

```
需求分析 → Prompt 设计 → AI 生成 → 人工审核 → 迭代优化
```

### 4.2 最佳实践

1. **任务分解**: 将复杂任务拆分为小步骤
2. **质量把控**: AI 输出必须人工验证
3. **知识积累**: 建立自己的 Prompt 库
4. **工具组合**: 结合多种 AI 工具优势

### 4.3 常见场景

| 场景 | AI 应用 | 注意事项 |
|------|---------|---------|
| **代码开发** | 代码生成、Bug 修复 | 需理解生成逻辑 |
| **内容创作** | 文章写作、文案优化 | 保持个人风格 |
| **数据分析** | 数据清洗、可视化 | 验证数据准确性 |
| **学习辅助** | 概念解释、习题解答 | 交叉验证答案 |
